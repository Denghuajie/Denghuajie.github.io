升级CoCoaPods之后的格式变动问题

发表于 2016-06-28   |     |   阅读次数

之前多个target的Podfile文件是用link_with target1, target2…表示添加多个target

在升级pod到1.0.0之后发现还这么写就报错：

查了下，现在改成用target ‘myTarget’ do了，所以正确的姿势应该是：

source ‘https://github.com/CocoaPods/Specs.git‘

platform :ios, ‘7.0’

def myPods
pod ‘MBProgressHUD’, ‘~> 0.9.2’
pod ‘AFNetworking’
pod ‘ReactiveCocoa’, ‘2.5’

end

target ‘Rac’ do
myPods
end

target ‘Rac_dev’ do
myPods
end

iOS-RichTextEditor

最近项目中有个需求是这样的：用户编辑日志，包含文字和图片，并可保存草稿或编辑，效果类似下图所示

这是豆瓣的写日记功能，可以看到文字与图片是一起排版的，同时默认插入图片的时候是往下换行。

研究这个需求的时候，最后想到的实现思路无非是两种：

  • 用html编辑，这个只需要将前端同事的页面跟js文件嵌入，同时做React与Native端的交互，上传相片的时候OC将图片url参数传给js。

  • iOS7之后苹果推出了TextKit，使用该框架将textView里的图片文字经过转义得到一个字符串，并传给后台。从服务器获取数据时在进行解析得到NSAttributedString交给textView，这个过程分上传与解析。

（在这里猜测豆瓣应该是用的第一种方式）

当时前端同事还没开始做这块，自己先用第2种方案实现，过程如下。

  • 如果是文字，直接输入即可。

  • 如果是图片，选择完图片的回调API里，将图片上传到服务器，拿到图片的url，然后再将图片插入textView。那么这里就有一个关键问题，在上传给服务器的字符串里，如何用相应的字符串表示图片？
