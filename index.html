通常我们遍历数组看起来像是这样（使用for循环）：

如果换成遍历字典或者set，那么复杂点：

  • 字典
  • Set

由于字典与Set都是无序的，不能直接用特定的下标去集合里取值，所有要先获取字典的所有key与Set的所有object，在获得的有序数组上遍历，以访问集合里的值。创建这个附加数组会有额外的开销，同时多创建了一个数组对象。

利用Objective-C的枚举器来遍历可以避免创建这种中介数组：

升级CoCoaPods之后的格式变动问题

之前多个target的Podfile文件是用link_with target1, target2…表示添加多个target

在升级pod到1.0.0之后发现还这么写就报错：

查了下，现在改成用target ‘myTarget’ do了，所以正确的姿势应该是：

source ‘https://github.com/CocoaPods/Specs.git‘

platform :ios, ‘7.0’

def myPods
pod ‘MBProgressHUD’, ‘~> 0.9.2’
pod ‘AFNetworking’
pod ‘ReactiveCocoa’, ‘2.5’

end

target ‘Rac’ do
myPods
end

target ‘Rac_dev’ do
myPods
end
